Fundamentals

This is documentation on the Gaming Docs Content Publisher Tool (CPT)

Media Queries

The CPT is built with the Bootstrap framework and includes a responsive 12-column fluid grid layout. The breakpoints below determine which layout styling is rendered in the browser. CSS declarations can be media-specific by wrapping them in the @media syntax:

@media screen and (media features) and (additional features) { [media specific CSS] }

Example:

@media only screen and (max-width:1024px) and (min-width:768px) { ... }

Breakpoints

$mobile-screen: emCalc(320px); $mobile-portrait: emCalc(480px); $mobile-large-screen: emCalc(560px); $small-screen: emCalc(768px); $tablet-screen: emCalc(1024px); $medium-screen: emCalc(1280px); $large-screen: emCalc(1440px); $ps4-fullscreen: emCalc(1900px);

Media Query Rules - variables must be wrapped in '#{$...}'

$mobile: "only screen and (min-width:'#{$mobile-screen}')"; $small: "only screen and (min-width:#{$small-screen}))"; $medium: "only screen and (min-width:#{$medium-screen})"; $large: "only screen and (min-width:'#{$large-screen}')";

Unique Identifiers

Every element in the CVA includes a unique identifier, or slug, assigned in a CSS class. Each class is named by the element type + sequential order of element on the current page, then nested after each parent element and always starting with the chapter.

Example

.ch-1-pg-1-bh-1-img-1

These unique IDs were introduced as a way to style specific instances of an element type without affecting the rest of its siblings. Use the unique class as a direct or parent selector in any CSS rule.

Type

The default font for the CVA is "SST", a non-standard font, and referenced by the @font-face rule. The base font size is 16px which we use to determine 1rem (root em) unit. All numerical values should be written in rem units, which are relative to the root or base font size as opposed to em units, which are relative to its parent and can cause compounding issues.

We currently use four fonts out of the "SST" font-set:

@font-face { font-family: "sstregular"; src: "..."; #relative path of font file font-weight: normal; font-style: normal; } @font-face { font-family: "sstitalic"; src: "..."; font-weight: normal; font-style: normal; } @font-face { font-family: "sstbold"; src: "..."; font-weight: normal; font-style: normal; } @font-face { font-family: "sstbold_italic"; src: "..."; font-weight: normal; font-style: normal; } body { font-family: "sstregular", Helvetica, Arial, sans-serif; }

Default font-size variables based on 16px base-font-size:

$font-size-small: 0.875rem; $font-size-normal: 1.15rem; $font-size-medium: 1.5rem; $font-size-large: 1.875rem; $font-size-larger: 3.25rem;

All text is rendered in the "SST" font in the base template, but can be replaced by theme overrides by uploading the font files (in .ttf, .woff, .eot and .svg formats) in the CPT then referencing the font(s) with the above syntax in the custom CSS overrides.

Icons

There is a custom set of icons rendered as SVG fonts and used throughout the CVA for pagination, expand, video and various other buttons. The font-family name is "psdoc-icons" which following CSS assigns to any class name that contains "icon-"

[class^="icon-"], [class*=" icon-"] { font-family: "psdoc-icons"; font-weight: normal; font-style: normal; text-transform: none; -webkit-font-smoothing: antialiased; text-decoration: inherit; display: inline-block; }
