Styling

Following are common styles used throughout the CPT.

Content Area

Content areas include forms, accordian panels, <tbody> and any other block of content and consistently share a light grey background-color of #f2f2f2. Add a .content-area class to any block element that contains content.

Markup Example

.content-area p This is some content in a content area.

Result

This is some content in a content area.

Headers

Most headers are contained within a div with a class of .header-row that sets a padding of 1rem (16px), a top/bottom margin of 1rem and a background-color of #e6e6e6

SCSS

.header-row { background: $light-grey; padding: 1rem; margin: 1rem 0; vertical-align: middle; }

Page Headers

Page headers are <h1> elements wrapped inside a .header-row and used to display the project name always at the top of the editor.

Jade

.header-row h1 | {{project.project_name}}

Result

Project Name

Section Headers

Section headers are <h3> elements wrapped inside a .header-row and used to display the titles of each section in the editor (eg. Upload Assets)

Jade

.header-row h3 | Upload Assets

Result

Upload Assets

Theme Section Headers

Headers within theme settings are stand-alone <h3> elements used to display titles of each theme setting (eg. Chapter Header Background). No container or class name needed.

Spacing

Custom CSS classes can be added to elements to add left and/or right margins and/or padding. Do not append the these classes to the grid column classes (.col-md-*) or the layout will break.

SCSS

// adds a left padding of 5px .pad-lt { padding-left: 5px; } // adds a right padding of 5px .pad-rt { padding-right: 5px; } // adds a left margin of 0.5rem (8px) .mrg-lt { margin-left: 0.5rem; } // adds a right margin of 0.5rem (8px) .mrg-rt { margin-right: 0.5rem; } // adds a bottom margin of 0.5rem (8px) .rowspace { margin-bottom: 1rem; }

Markup Example

.col-md-3 button.btn.btn-default.mrg-rt i.fa.fa-pencil.pad-rt | Edit

Alignment

Use these classes to align your content within the grid system.

Text-align

Custom CSS classes can be added to elements including grid column classes to align text right or center; the default text alignment is always left.

SCSS

// aligns text to the right .align-right { text-align: right; } // aligns text center .align-center { text-align: center; }

Markup Example

.col-md-3.align-right p This is right-aligned text

Result

This is right-aligned text

Floats

Use these native Bootstrap classes to float elements left or right in its parent container.

SCSS

.pull-left { float: left; } .pull-right { float: right; }

Markup Example

.col-md-3 p.pull-right This is a floated div

Result

This div is floated right

Remember to clear your floats if they are not already contained in a block element. Apply .clearfix to the container of your floated element to clear the floats.

.clearfix { @include clearfix(); }

Buttons

Each of the seven button styles throughout the framework represent a different action. Most instances of buttons are initiated with the button tag appended by a class of .btn and button type class .btn-* defined below.

.btn-default
.btn-primary
.btn-info
.btn-success
.btn-warning
.btn-danger
.btn-inverse

Markup Example

button.btn.btn-primary I'm a primary button

Result

Tabs

Tabs are used in the Theme Settings to show the different categories of theme styling for a project, but can be used elsewhere if necessary. The custom CSS includes a normal, hover and active states on right-aligned tab buttons.

Markup Example

.settings-tabs button.btn.btn-tab Tab 1 button.btn.btn-tab Tab 2 button.btn.btn-tab Active Tab

Result

Button Containers

Most buttons are right-aligned within the CPT. Wrap buttons within a .btn-container div to achieve this style.

Markup Example

.btn-container button.btn.btn-primary Action Button

Result

Theme Actions

Action buttons grouped at the bottom of a section or page are are grouped together in a .btn-container div nested inside a .theme-actions div that gives it a $light-blue background-color.

Markup Example

.theme-actions .btn-container button.btn.btn-primary Action Button

Result
